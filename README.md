# Итоговый проект

------------


### Создание и активация виртуального окружения:

- **sudo apt install python3-virtualenv**
- **virtualenv venv**
- **source venv/bin/activate**

### Установка зависимостей:

- **pip install poetry**
- **poetry install**

### Вводная информация:

Т.к. у нас нет регистрации, то при запуске сразу создаётся тестовый пользователь с api-key: test, и если введённый api-key не существует, то программа автоматически переключит на тестового пользователя.

### Запуск:

Если мы запускаем в режиме разработки, то в my_twitter/app/database/ есть docker-compose.yaml для БД, запускаем сначала его.

В **my_twitter/settings.py** есть настройки, их лучше не трогать, проект сам определит, где Вы его запускаете, на компьютере, в тестах или в докере и подготовит всё, что бы эти запуски не пересекались.
Итак запуск:

- **uvicorn my_twitter.main:app --reload --reload-exclude database**

database - эта папка появится, когда мы запустим контейнер докера, туда будет проброс данных из БД, поэтому мы её исключаем из отслеживания. При запуске возможно даже увидим это: "PermissionError: Permission denied...python_advanced_diploma/database/db...", не обращаем на это внимание, т.к. у uvicorn нет прав заглядывать туда.
Если ёё нет, то просто выполняем это:

- **uvicorn my_twitter.main:app --reload**

Для запуска контейнеров, выполним это:

- **docker compose up**

### Создание пользователей.

Для создания пользователя выполним следующее:

- **python manage.py createuser**

И следуем сообщениям. Можно выполнять многократно.

Для создания сразу несколько пользователей, например для демонстрации, выполним следующее:

- **python manage.py createuser N**

Где N - это количество пользователей, в ответ получим их api_key. Выполнить можно только раз.
